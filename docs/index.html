CloneVoice API (1.0.0)

Download OpenAPI specification:

Voices, languages, audio generation, and generated audio retrieval

The CloneVoice API provides endpoints for managing voices, languages, audio generation, and retrieval of previously generated audio.

Main capabilities:

  • Voices: list & fetch voices (system, cloned, designed, public library)
  • Languages: list available languages
  • Audio: generate audio from text and check audio status
  • Generated Audio: list and retrieve generated audio with rich filtering

Voices

Manage and retrieve voices

Get all voices

Retrieves all voices for the authenticated user with optional filtering and pagination. This includes user's voices (cloned, designed, public_library) and system voices.

Authorizations:
ApiKeyAuth
query Parameters
category
string
Enum: "system" "cloned" "designed" "public_library"

Filter by voice category.

search
string

Search in voice name, description, or label.

gender
string

Filter by gender (male, female, others). Supports multiple values, comma-separated, e.g. gender=male,female.

age
string

Filter by age group.

favorite
boolean

Filter by favorite status.

  • true: only favorite voices
  • false: only non-favorite voices
exported
boolean

Filter by exported status.

  • true: only exported voices
  • false: only non-exported voices
system
string
Default: "yes"
Enum: "yes" "no"

Include system voices (yes or no).

sort_by
string
Default: "created_at"
Enum: "name" "created_at" "updated_at" "category" "gender" "age"

Sort field.

sort_order
string
Default: "desc"
Enum: "asc" "desc"

Sort order (asc or desc).

per_page
integer [ 1 .. 100 ]
Default: 20

Number of items per page (1–100).

page
integer >= 1
Default: 1

Page number (1-based).

Audio

Generate audio from text and check status

Generate audio

Generates audio from text using a selected voice and language.

The process is asynchronous: generation happens in the background. You can either provide a webhook URL to receive notifications, or poll the GET /audio/{id_or_uuid} endpoint.

Authorizations:
ApiKeyAuth
Request Body schema: application/json
required
script
required
string <= 7000 characters

Text to convert to audio.

voice_id
required
integer

ID of the voice to use.

language_id
required
integer

ID of the language.

title
string or null <= 255 characters
webhook_url
string or null <uri>

URL to receive webhook notifications.

high_similarity
boolean
Default: true

When true (default), the audio generation prioritizes higher similarity to the selected voice. Set to false to use standard similarity mode.

Responses

Production

https://app.clonevoice.ai/api/v1/audio

Staging
